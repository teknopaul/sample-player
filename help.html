Installation

The sampler is written as a single web page, simply extracting the zip file and double-clicking the index.html file will load the page in a browser. It is designed for FireFox, but should work in a HTML5 compliant browser. Don't bother to send me you Internet Explorer woes, go get a proper browser.

Key bindings

The sampler was designed to work with an external USB calculator keyboard which are available for laptops that dont have an in built numeric keypad. A normal keyboard works fine.

The web page must be focused, not in the background. Click anywhere on the page with the mouse if you find the keys are not responding.

The 1-9 keys and the 0 key from the number pad or the main keyboard are used to trigger samples.

The space bar can be used to stop all sounds.

Pressing alt + [number key] can be used to stop only one of the samples.

The Up ↑ and Down ↓ keys are used to swap between the sample banks,
N.B. this can be achieved on the numeric keypad by temporarily disabling numlock.

Other normal browser keys are enabled such as F11 to go full screen.

Editing soundbanks

Editing sound banks is done by creating directories of up to 10 .wav files and editing JSON files with a text editor.

There are no GUI tools to edit the JSON because HTML5 apps dont have very good access to the filesystem. I might one day writea FireFox plugin to enable editing or a downloadable/installable "app". Get in contact if you think that would be worth while.

banks.json

The banks.json file should be in the same directory as this help file and the index.html file that renders the smapler. Its contents are simply a list of strings which represent subdirectories directories of wave files. 

[
	"act1-sound-bank",
	"act2-sound-bank"
	"act3-sound-bank"
]
In thge above example 3 directories, 3 directories called ./act1-sound-bank, ./act1-sound-bank, ./act1-sound-bank should be created containing the .wav files and a sound-bank.json file should be created in each directory.

sound-bank.json

A sound-bank.json file contains primarily an array called samples defining the .wav files and also other meta data currently just the background-color which can be set to make it obvious which sound bank is loaded. 


{
	"name" : "Drums Bank",
	"background-color" : "#eef",
	"samples" :  [
		{"name" : "Boom",   "mediaType" : "audio/wav", "url" : "boom.wav" },

		{"name" : "Kick", "mediaType" : "audio/wav", "url" : "club-kick.wav" },
		{"name" : "HH", "mediaType" : "audio/wav", "url" : "hi-hat.wav"     },
		{"name" : "HH",   "mediaType" : "audio/wav", "url" : "hat.wav" },

		{"name" : "Kick",   "mediaType" : "audio/wav", "url" : "kick-snare.wav" },
		{"name" : "Scratch",   "mediaType" : "audio/wav", "url" : "record-scratch.wav" },
		{"name" : "Cymbal",      "mediaType" : "audio/wav", "url" : "cymbal.wav" },

		{"name" : "Kick",   "mediaType" : "audio/wav", "url" : "kick.wav" },
		{"name" : "Snare", "mediaType" : "audio/wav", "url" : "tape-snare.wav" },
		{"name" : "Cymbal",  "mediaType" : "audio/wav", "url" : "soft-cymbal.wav" }
	]
}
